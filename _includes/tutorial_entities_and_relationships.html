Entities and relationships

This tutorial walks you through the process of writing the ORM code for an imaginary blog application. It illustrates the main features of the library.

The blog application consists of users that are allowed to write posts to be published on a website. The posts can be associated with any number of pre-existing categories. Users have profiles with some personal information.

This gives us the following entities :

  • User : id (pk), login, password
  • Profile : id (pk), email
  • Post : id (pk), content, user_id (fk of user)
  • Category : id (pk), name
  • Category2post (associative entity) : category_id (pk, fk of category), post_id (pk, fk of post)
