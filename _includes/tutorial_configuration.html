Configuration

None, because the following conventions have been respected :

  • table names = pluralized entity names,
  • foreign keys = primary keys prefixed with entity name followed by '`_`' (entity user : pk = 'id' => fk = 'user_id'),
  • associative entity name = <name of 1st entity>2<name of 2nd entity> (or <name of 2nd entity>2<name of 1st entity> : whichever is first by alphabetical order comes first),
  • tables are stored in 'default' database,
  • entity field names = model property names= database column names,
  • ...and for now we're happy with model objects being simple stdClass instances.

So if you've created the above tables, you can start playing right away with the rest of the tutorial !
