Deleting data

Censoring the content of John's post is not enough after all, we want him to feel the full extent of our wrath and so we decide to delete his post entirely.

Assuming we already have a model object available for that post, we could do it like this :

glue::delete('post', $post);

Or, if we had more than one post to delete :

glue::delete('post', array($post1, $post2));

If we wished to delete some posts but we don't have the model objects at hand, we may still delete them by passing the primary key to the delete function :

$post = glue::delete('post', 3);

...the function will return the deleted post (with only the primary key loaded), or null if no such post was found./p>

An array of conditions can also be used :

$posts = glue::delete('post', array('id' => array(2, 3, 4))); // means "where id IN (2, 3, 4)"
$posts = glue::delete('post', array('user_id' => 2, 'content' => 'test')); // means "where user_id = 2 AND content = 'test'"

...in this case the function will return the set of deleted objects, which may be empty if no entity instances match the criteria in the database.

This method is NOT intended for mass deletion of data, since the objects will be loaded first before being deleted. Always use raw SQL for mass deletion, which you may put in a custom function in the corresponding entity mapper.
