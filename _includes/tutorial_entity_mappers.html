Entity mappers

Although you won't have to deal with entity mappers directly, OGL makes use of them everywhere behind the scene. So before you dive deeper into this tutorial, it's better to have a basic understanding of what they are used for.

Entity mappers are objects that describe how entity instances are represented in the model and persisted into the database. Although it is possible to define your own mapper classes if you need something out of conventions, in this tutorial we'll let OGL instantiate the default mapper class.

You may get the mapper object for the 'post' entity like this :


$mapper = OGL::entity('post');

If you var_dump it, you'll see how OGL initialized it by database introspection :


protected $name   => 'post'                             // Entity name
protected $db     => 'default'                          // Database it is stored into
protected $model  => 'stdClass'                         // Model class
protected $tables => array ('posts')                    // Tables the entity is stored into
protected $fields => array ('id', 'content', 'user_id') // Fields identifiers
protected $properties => array (                        // Mapping fields => object properties 
    'id'      => 'id',
    'content' => 'content',
    'user_id' => 'user_id'
  )
protected $columns => array (                           // Mapping fields => databases tables/columns
    'id'      => array ('posts' => 'id'),
    'content' => array ('posts' => 'content'),
    'user_id' => array ('posts' => 'user_id')
  )
protected $types => array (                             // Mapping fields php types
    'id'      => 'int',
    'content' => 'string',
    'user_id' => 'int'
  )
protected $pk => array ('id')                           // Fields in the primary key
protected $fk => array (                                // Mapping fields in the pk => fk identifiers
    'id' => 'post_id'
  )
protected $autoincrement => boolean true                // Whether or not pk is auto-incrementing
