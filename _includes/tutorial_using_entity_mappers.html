Using entity mappers

If you have a lot of things to do with some entity, you might find it more convenient to get the mapper for that entity and to directly call it rather than using Glue's static functions as gateways.

So instead of this...

$post = glue::create('post', array('content' => "some content", 'user_id' => $user->id));
glue::insert('post', $post);
glue::update('post', $post);
glue::delete('post', $post);
$post = glue::select('post', 3);

...you might do that :

$mapper = glue::entity('post');

$post = $mapper->create(array('content' => "some content", 'user_id' => $user->id));
$mapper->insert($post);
$mapper->update($post);
$mapper->delete($post);
$post = $mapper->select(3);

...so you don't have to pass the 'post' parameter each time you call a function.
