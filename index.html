What's Glue ?

Glue is an Object Relational Mapping library for PHP. It is currently distributed as a module for the Kohana PHP framework. Glue is based on Data Mapper, which differs from the more popular pattern that is Active Record. This means model classes don't have to extend a special base class and objects called mappers are used to load, validate and save model objects.

Notable Features

  • ORM-free model classes : model classes are yours entirely. Glue will never get in the way of what you want to do there. It is nothing more than a helper to load and save object graphs.
  • Minimal number of database queries : specifically, the number of database queries doesn't skyrocket when loading N model objects and for each of them a collection of many-to-many or one-to-many related objects.
  • No configuration : you don't have to define mappers, model classes or relationships to start playing with this. Default ones are instanciated if you don't provide yours and database introspection is used to guess entity field names and types.
  • Single model class, multiple tables mapping : a model class can be mapped to several tables joined together by primary key, which is useful for object-relational mapping of inheritance.

Installation

Same as any Kohana module. Copy the Glue directory into your modules folder and add this line to bootstrap.php in your application folder :

Kohana::modules(array(
		...
		'database' => MODPATH.'database', // Required
		'glue' => MODPATH.'glue',
	));

Documentation

There is a tutorial and some more philosophical documentation. I strongly recommend reading the tutorial first before diving into the documentation. It is more user friendly and has a better chance of staying up-to-date with the code.
